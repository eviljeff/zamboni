{{ _('Steps') }}

{%- trans %} Submitting an app to the Firefox OS Marketplace involves the following high-level steps: {% endtrans -%}

  1. {{ _('Log in to the Firefox OS Marketplace site.') }}
  2. {{ _('Enter information about the app.') }}
  3. {%- trans %} Optionally, set pricing, payment, and refund options. {% endtrans -%}

{{ _('Log in to the Firefox Marketplace site') }}

  1. {%- trans %} In a Web browser, visit the Firefox Marketplace site. {% endtrans -%}
  2. {%- trans %} At the bottom of the page, click Developer Hub. {% endtrans -%}
  3. {%- trans %} Click the Login/Register link at the top of the page. A login window opens. {% endtrans -%}
  4. {{ _('Login with Mozilla Persona:') }}
    • {%- trans %} If you've previously used Persona, select the email address for the identity you want to use, and click sign in. {% endtrans -%}
    • {%- trans url='//support.mozilla.org/kb/what-is-persona-and-how-does-it-work' %} If you haven't used Persona, or you want to use a different email address, you need to create an account. {% endtrans -%}

    {%- trans %} The Persona window closes, and you are logged in to the Marketplace Developer Hub. {% endtrans -%}

{{ _('Enter information about the app') }}

  1. {{ _('Click the Submit An App button.') }}
  2. {%- trans %} Review the Developer Agreement and click Agree and Continue. {% endtrans -%}
  3. {%- trans %} Choose whether your app will be Free or Paid. {% endtrans -%}
  4. {%- trans %} Select the devices which your app is compatible with. {% endtrans -%}
  5. {{ _('Provide the location of your app') }}
    1. {%- trans url='//developer.mozilla.org/docs/Apps/Manifest' %} If your app is a web app, enter the location on your server of the app manifest for your app (this must be a full URL), and click Validate. {% endtrans -%}
      • {%- trans %} If the app manifest validates, click Continue. {% endtrans -%}
      • {%- trans %} If there are validation errors, correct them and revalidate the manifest. {% endtrans -%}
    2. {%- trans %} If your app is a packaged app, click the Packaged tab and select the ZIP file that you wish to upload. {% endtrans -%}
      • {%- trans %} The package will upload automatically. If your app validates, click Continue. {% endtrans -%}
      • {%- trans %} If there are validation errors, correct them and re-upload your package. {% endtrans -%}
  6. {%- trans %} Add the required details for your app's detail page, and any other details you want to add, and click Continue. Note that the required fields are the brief summary, categories, privacy policy, device types, and a screenshot. But a homepage, support site, and support email are highly recommended. The multi-line text fields support HTML, so you can structure and link the text. {% endtrans -%}

{%- trans url='#after-submitting' %} If you are charging money for your app or for items within your app, continue to the next section. If you are distributing your app for free, skip to After you submit the app. {% endtrans -%}

{{ _('Set pricing, payment, and refund options') }}

{%- trans %} Note: The Firefox OS Marketplace does not currently support paid apps or in-app payments. We hope to enable this feature for testing late in the first quarter of 2013. {% endtrans -%}
