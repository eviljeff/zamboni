{{ _('Hosted Apps') }}

{%- trans %} Once you have finished your app, you need to do the following: {% endtrans -%}

  • {%- trans %} Deploy the app — Arrange for web hosting and get the app up and running on the web server. {% endtrans -%}
  • {%- trans %} Publish the app — Make the app available for end users to install. {% endtrans -%}

{{ _('Deploying the app') }}

{%- trans %} You need to host the app on a publicly accessible Web server, just like any other website. There are numerous ways to do this. {% endtrans -%}

{{ _('GitHub') }}

{# L10n: Do not localize text in #} {%- trans url_github='http://pages.github.com', url_mdn='//developer.mozilla.org/docs/Apps/Manifest#Serving_from_GitHub' %} If the Web app is purely static (HTML/CSS/JavaScript, but no server-side processing), GitHub Pages is a solid hosting option. It will serve your manifest with the correct MIME type if you give it a .webapp extension. {% endtrans -%}

{{ _('Generic hosting solutions') }}

{%- trans url_heroku='http://www.heroku.com', url_google='http://code.google.com/appengine' %} For dynamic websites, use a generic hosting option (like a Web server you may already own or have access to) with the right capabilities or a hosting provider specifically tailored to the needs of your app, like Heroku or Google App Engine. {% endtrans -%}

{{ _('Publishing the app') }}

{%- trans %} Once your Web app has been deployed, you obviously want to make it installable by your intended user base. There are currently two options for publishing your app. {% endtrans -%}

{{ _('Submit to Firefox Marketplace') }}

{%- trans url='//developer.mozilla.org/docs/Apps/Submitting_an_app' %} Submitting your app to the Firefox Marketplace has many advantages, such as increased publicity, no need to implement special APIs on your own Web site, and the possibility of publishing paid apps more easily. {% endtrans -%}

{{ _('Publish on your own website') }}

{%- trans url='//developer.mozilla.org/docs/Apps/Apps_JavaScript_API' %} If you want to let people install your app from a website you own, you must implement some JavaScript code on your Web site to manage installing and updating your app into users' browsers. {% endtrans -%}
